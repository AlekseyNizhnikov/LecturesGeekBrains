# Моё руководство по работе с Git

## Семинар 1 - Команды терминала:

1. **Git init** - команда терминала, которая позволяет инициализировать в нужной дирректории файлы, необходимые для работы системы контроля версий.

2. **Git version** - команда терминала, которая выводит на экран информацию о версии установленной системы контроля ревсий Git, а также является сигналом о правильной и корректной установки Git.

3. **Git config --global user.name "Name"** - команда терминала, которая позволяет установить имя пользователя под которым будут выполнятся комиты.

4. **Git config --global user.email "E-mail"** - команда терминала, которая позволяет установить адрес электронной почты, для связи с пользователем, который создал комит (см. п. 3).

5. **Git add .\FileName.txt** - команда терминала, которая позволяет добавить файл или директроию в рабочее пространство системы контроля версий. Частный случай команды является: git add * - она добовляет в git ВСЕ файлы которые есть в данной папке.

6. **Git commit -m "Comments"** - команда терминала, которая позволяет зафиксировать разность между текущей версией файла и более ранней, закомиченной. Каждый коммит сопровождается обязательным комментарием, лаконично описывающим произведенные за данный коммит действия.

7. **Git log** - команда терминала, которая выводит на экран список всех зарегестрированных коммитов. Так же выводится дата, время, имя пользователя git config и e-mail, введеные командой git config (см. п. 3).

8. **Git status** - команда терминала, которая выводит список файлов которые расположены в папке инициализации git (см. п. 1), но не добавленные в рабочее пространство (не проиндексированы) (см. п. 5), либо удаленные из рабочего пространства системы контроля версий.

9. **Git diff** - команда терминала, которая выводит разность между текущим состоянием файла и файлом закомиченым ранее.

## Семинар 2 - Команды терминала. Создание веток и работа с ними:

1. **Git branch name_branch** - команда терминала, которая позволяет создать еще одну ветку состояния репозитория, например для выгрузки заказчику проекта по-частям, либо для параллельной работы над одним проектом в команде.

2. **Git checkout name_branch** - команда треминала_ .которая позволяет перейти к нужной ветки в дереве коммитов, либо к конкретному коммиту, указав его номер или первые 5 - 6 символов кода.
    * Если записать команду таким образом: git checkout name_branch^, то получится что мы чекнимся к нужному коммиту, без указания его кода (хеша).
    * Если записать команду таким образом: git commit name_branch^^, то переместимся к прародителю коммита.
    * Если нужно указать конкретное число коммитов, то заменяем знак ^ на ~numbers
    * С помощью оператора ^ можно принудительно привязывать какую либо ветку к конкретному коммиту, указывая через тильду, номер этого коммита: git branch -f name_branch HEAD~4. HEAD используется как указатель на конкретный коммит.

3. **Git branch -d name_branch** - команда терминала, которая позволяет удалить ненужную ветку.

4. **Git merge name_branch** - команда терминала, которая позволяет выполнить слияние двух веток. К ветке в которой находится пользователь, добавляется ветка, указанная в команде (name_branch).

5. **Вставка изображения** - ![Вставка изображения](Images_one.jpg)

6. **Git branch** - команда терминала, кторая позволяет посмотреть весь список веток репозитория, с указанием ветки в которой находится пользователь.

7. **Clear** - команда терминала, которая очищает историю терминала, делая окно чистым.

8. **Git commit -amend -m "New commit"** - команда терминала, которая позволяет изменить последний коммит. Если нужно изменить конкретный коммит, переходим на более поздний и применяем данную команду.

9. **Git reset name_commit** - команда терминала, которая возвращает нас на шаг назад. Смещает master на коммит назад.

11. **Git checkout -b name_branch** - команда терминала, которая создает новую ветку и тутже туда переходит.

12. **Git rebase name_branch** - команда терминала, которая позволяет скопировать все коммиты из текущей ветки в ветку name_branche, создавая при этом в ней новый коммит. Чтобы ветка с которой мы слились поднялась до уровня ветки из которой мы слили, необходимо сменить ветку и применить ту же функцию. Эта команда делает дерево коммитов более линейным, лаконичным и понятным.

13. **Git revert HEAD** - команда терминала, которая позвляет отменить все предыдущие коммиты и вместо них создать один единственный коммит включающий изменения всех предшествующих. Не изменяет истории коммитов.

14. **Git reset HEAD~numbers** - команда терминала, которая позволяет отменить действие предыдущего коммита и вернуться назад. Назад в будущее....

15. **Git cherry-pick <commit> <commit>...** - команда терминала, которая позволяет скопировать конкретные коммиты в то место где мы находимся (HEAD).

16. **Git rebase -i HEAD~4** - команда терминала, которая в текстовом редакторе откроет список последних 4 коммитов и их хеши. После чего их можно будет объеденить, удалить, переместить. Далее будет создана отдельная ветка с выбранными коммитами, относительно HEAD.

17. **Git tag name_tag** - команда терминала, которая позволяет создать постоянную и неизменную ссылку на конкретный коммит.

18. **Git describe <ref>** - команда терминала которая показывает насколько далеко мы от ближайшего тега (см. п. 17). Ref это любая ссылка на коммит, если ее не указать то git будет считать что имеют ввиду HEAD. Вывод следующий: tag_numbersCommit_hash.

## Семинар 3 - Команды терминала. Удаленный репозиторий:
   
1. **Создание удаленного репозитория на GitHub** возможно 3 тремя способами:
* Если у нас нет локальных файлов, которые можно отправить на сервер GitHub... Тогда поочередно вбиваем в териминал команды представленные ниже:
    + echo "# qwert" >> README.md
    + git init
    + git add README.md
    + git commit -m "first commit"
    + git branch -M main
    + git remote add origin https://github.com/AlekseyNizhnikov/qwert.git
    + git push -u origin main

* Если локальные файлы есть, тогда выполняем следующие операции:
    + git remote add origin https://github.com/AlekseyNizhnikov/qwert.git
    + git branch -M main
    + git push -u origin main
* Если требуется импортировать код из другого репозитория (видимо удаленного), пока этот момент не разобрали...

2. **Git remote add origin url** - команда терминала, которая позволяет перейти по ссылке url и создать там удаленный репозиторий, создав там копию текущего локального репозитория.

3. **Git branch -M main** - команда терминала, которая создает в удаленном репозитории основную ветку, под названием main.

4. **Git push -u origin main** - команда терминала непосредственно отправляющая файлы на сервер. При отправки первый раз может потребоваться авторизация. Если main изменен в пункте 3., то здесь его требуется также изменить. В дальнейшем, после авторизаци пулл, можно делать простой командой: git push.

5. **Git pull** - Если требуется выгрузить с сервера актуальную верси репозитория, используется команда git pull. Выполняя ее мы подтягиваем более свежие файлы с сервера. ТАкое может потребоваться, если, например, изменения производились другим человеком, либо с другого компьютера, либо в самом GitHub.

6. **Материал для справки по Git** - https://vimeo.com/showcase/5616060

7. **Pull requests** - особенность систем контроля версий, которая позволяет выполнить некоторые операции с удаленным репозиторием, а затем предложить свои изменения владельцу исходных файлов - отправить запрос. Актуально для проектов с открытым кодом или при работе в команде.

8. **Локальный репозиторий** - если требуется скачать только локальную версию репозитория, то в GitHub предусмотренна соответствующая зеленая кнопка: Code -> HTTPS которая дает ссылку на прямое скачивание файлов.
